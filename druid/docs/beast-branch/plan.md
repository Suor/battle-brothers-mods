# Beast Branch — дизайн-план

Статус: дизайн (не реализация). Альтитуда — *что* и *почему*, без чисел статов и кода:
конкретные значения помечены как открытые вопросы.

## Цель

Разбить дерево перков друида на две взаимоисключающие роли, чтобы у бэкграунда было не один
оптимальный билд, а несколько различимых. Ведущий принцип (из `user-requests.md` §8):
**меньше синергирующих перков — лучше.** Разнообразие билдов рождается из развилок и из
конкуренции кастомных перков с обычными боевыми, а не из длинной цепочки синергий (болезнь
necro). Поэтому группы держим небольшими (по ~3–4 кастомных перка).

## Структура (две группы, не дерево)

Перки в BB независимы — это не дерево с ветками. Используем взаимные исключения двух
**групп**: Supporter и Fighter. Выбор группы **неявный** (§19): нет перка-«ворот» у Supporter —
группу определяет первый взятый групповой перк.

- **Взятие любого Supporter-перка** (`Regrowth`, `Entangling Roots`, `Hatch`, `Apex`)
  блокирует `Beastform` — а с ним и весь Fighter (`Beast Aura` и `Beast Rage` гейтятся
  самим `Beastform`).
- **Взятие `Beastform`** блокирует все четыре Supporter-перка.
- Внутри группы перки **независимы** — берёшь любые по мере открытия тиров, без порядка.
- `Venom` — вне групп, доступен всегда; работает по-supporter, пока не взят `Beastform`.
- Обоснование: Supporter — это два под-стиля (призыватель и хилер-контроллер), и любой
  одиночный перк-ворота облагал бы один из них «налогом» на ненужный перк. Неявный выбор
  даёт вход через любой стилеобразующий перк, сохраняя взаимоисключение ролей.
- **UX-требование:** в тултипах групповых перков явно писать «закроет путь Зверя» /
  «закрыто путём Природы»; для заблокированных есть `IconDisabled`.

```
Общее:       Summon Beast (не перк, стартовая активка)
             Venom        (адаптивный; до Beastform — supporter-стиль)

Supporter:   Regrowth, Entangling Roots, Hatch, Apex Predators
             — первый взятый блокирует Beastform
Fighter:     Beastform (ворота ветки) → Beast Aura, Venom ⟷ Beast Rage
             — Beastform блокирует Supporter-перки
```

`Pack Leader` удаляется (см. §Beast Aura).

### Размещение по рядам (решено, §21, §23)
`Row` — 0-based индекс ряда дерева; уровень персонажа = Row + 2.

```
Row 1 (лвл 3):  Regrowth        | Beastform
Row 3 (лвл 5):  Hatch           | Beast Aura
Row 4 (лвл 6):  Entangle        | —
Row 5 (лвл 7):  —               | Venom ⟷ Beast Rage   (Venom вне групп)
Row 6 (лвл 8):  Apex Predators  | —
```

Venom и Beast Rage — на одном ряду: прямой одновременный выбор «берсерк vs хитрый боец»,
без смещения из-за разницы в доступности. На каждом ряду не больше двух перков (§23).
Venom поздно ещё и потому, что до лвл 5 призыв даёт только базового `Wolf` — травить
особо некому.

## Решённые элементы

### Выбор группы (неявный)
Группу определяет первый взятый групповой перк (§19, см. §Структура). `Beastform` берётся
осознанно под видимые статы; «плохие ролы → просто не бери Beastform» (это не ловушка,
а выбор).

### Совместимость (§20)
Как necro: работаем в чистой ванили — взаимные исключения реализуются собственной логикой
перков, без зависимостей от Reforged и Dynamic Perks; опциональная совместимость с ними —
по образцу necro (`facts.md`).

### Supporter
Сохраняет призывной/контрольно-лечащий набор: `Regrowth`, `Entangling Roots`, `Hatch`,
`Apex Predators`. **Под-стиль «не-призыватель»** допустим: можно пропустить `Hatch`/`Apex` и
играть от `Regrowth` + контроль + `Venom` на звере из слота-аксессуара (AC-пёс). Эта
асимметрия (Supporter может нести больше кастомных перков) намеренна.

### Beastform (пассив, разовое превращение)
Берётся один раз, эффект перманентный:
- **Лимиты снаряжения:** щит ≤ 10, шлем ≤ 10, броня ≤ 20, без дальнобоя. (Лёгкий
  щит/баклер ≤10 допустим — нужен для «щит на старте».)
- **Бонусы** (ориентир от пользователя, §15–16; точная подгонка — баланс-проход Q5):
  +10% Melee Skill, +10% Melee Defense, +20% HP. Регенерации нет — она в `Beast Rage` (§16).
- **Живучесть — внутри самого узла Beastform**, не отдельным перком: щита нет, а отдельный
  перк на броню стал бы авто-пиком (`user-requests.md` §7 «Thick Hide always-take», §8).
  Компенсацию запрета щита даёт встроенный бонус HP.

### Beast Aura (Fighter)
- **Аура-бонус** — любым союзным зверям (включая AC чужих братьев) в радиусе ~2 клеток:
  только мораль/бесстрашие — не убегают + бонус Resolve (§25); главная ценность перка —
  смена ИИ, не статы. Поглощает `Pack Leader`, но по близости, не глобально.
- **Смена ИИ твоих зверей** (призванного и выпущенного из слота): держатся рядом, не уходят в
  преследование. Не приказ — игрок зверями не управляет.

### Beast Rage (Fighter)
По образцу `perk_rf_feral_rage` из Reforged (`facts.md`), с двумя отличиями (§16):
- **Снижение урона заменено регенерацией:** 2 HP за ход за стак (вместо −2% урона за стак).
- **+1 стак за промах по врагу** — вдобавок к рефорджевским +2 за мили-попадание вплотную,
  +3 за мили-убийство, +1 за полученный удар; −3 стака в начале хода, сброс после боя.
- Остальное как у Feral Rage: за стак +2% мили-урона, +2 Resolve, +2 Initiative,
  −1 Melee Defense. Баффы и дебаффы копятся вместе — на старте боя ни ярости, ни штрафов.
- На пороге **~10 стаков** (§24; подгонка — Q5) — **сброс щита прямо в бою** + рык.
- **Опциональность = саморегуляция:** при плохой защите перк просто не берут.
- Эмерджентный выбор экипа: all-in без щита → двуручка; либо щит на старте → одноручка/dual-wield.

### Venom (адаптивный, отдельный перк)
Ослабляющий (не уроновый), не стакающийся дебафф. Вне групп, доступен всегда; до взятия
`Beastform` работает по-supporter. Значение зависит от группы:
- **Supporter:** травят твои звери (призванные и выпущенные из слота).
- **Fighter:** травишь сам — твои мили-атаки накладывают тот же poison-coat (§26);
  защитная ценность выходит из замедления/слепоты. Взаимоисключается с Beast Rage
  (берсерк vs хитрый боец).
- **Взят до `Beastform`, потом взят `Beastform`** (§17): Venom переключается на
  fighter-вариант, Beast Rage блокируется — ранний Venom означает неявный предвыбор
  «хитрого бойца».

### Pack Leader — удалить
Эффект «не бегут» уходит в `Beast Aura`. 

### Скейлинг зверей
До порога весь биомный пул заблокирован: призыв **всегда** даёт обычного `Wolf`, независимо
от биома (§22). Примерно на ур. 5 (ряд `Hatch`, Row 3) открывается полное биомное
разнообразие (direwolf, spider, …). `Apex` остаётся верхним порогом. Реализуемо подменой
сущности (`facts.md`: `wolf`/`direwolf`/`direwolf_high` — разные сущности). Скейлинг статов
зверей с уровнем / перки им на порогах — отложено в планы на будущее.

### Графика
Перманентный облик гуманоида-зверя (не животного) при взятии Beastform. Решено (§18): реюз
ассетов «Baku» из Fantasy Bros — мохнатое тело `bust_naked_body_7873/7874` + голова
`bust_head_7880–7882` + грива `hair_beast_01–05` (`facts.md`); механизм — подмена брашей
`setBrush()` по образцу `xxhero_baku_background.nut`. Подмена модели на лету не нужна — на
пороге Beast Rage добавляется только рык (звук) и сброс щита.
