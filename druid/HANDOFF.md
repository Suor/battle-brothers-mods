# Proper Druid - handoff

A new Battle Brothers mod (Modern Hooks + MSU + stdlib), modelled on Proper Necro. A **Druid**
background that summons beasts chosen by the battlefield biome (uncontrolled, faction
`PlayerAnimals`), entangles enemies, regenerates an ally, and grows via druid-only perks. Includes
an origin scenario and RU localization (Rosetta).

## Build
- `cd mods/druid && make check` → **currently green** (Syntax OK, Tests OK, Rosetta OK).
- `make install` (to `DATA_DIR` from `.env`), `make zip` (release).

## Code (implemented, compiles)
- `scripts/!mods_preload/mod_druid.nut` - entry + hooks: hireable via settlement draft lists (small/
  medium only, more in forest/lumber/swamp; never big cities); beasts give the druid XP/kills
  (FakeKill); actor master tracking; perk-tree injection (`data_helper`) + DynamicPerks group; JS/CSS.
- `scripts/skills/backgrounds/druid_background.nut` - the background.
- `scripts/skills/actives/druid_summon_beast.nut` - biome→beast (`::Const.Druid.Biomes` in
  `scripts/config/z_druid.nut`), 1/battle (+1 with Pack); spawns the enemy entity as PlayerAnimals;
  Pack Leader → `MoraleState.Ignore`; Venom → adds `druid_venom`; Apex → `makeApex()`
  (wolf→direwolf, hyena→hyena_high; spider/schrat_small/serpent grown in place: size+HP).
- `druid_entangle.nut` (inherits `root_skill` = game "Корни"), `druid_regrowth.nut` +
  `effects/druid_regeneration_effect.nut` (one carrier, no expiry), `racial/druid_venom.nut`.
- Perks: `skills/perks/perk_druid_{apex,pack,regrowth,entangle,venom,pack_leader}.nut`, defs in
  `config/z_druid.nut`, group in `druid_pg.nut`/`druid_load.nut`, UI `ui/mods/druid.{js,css}`.
- `scripts/scenarios/world/druid_scenario.nut` - origin. `druid/rosetta_ru.nut` - RU strings.

## Design decisions
- Uncontrolled beasts (PlayerAnimals); standalone (spawns the game's own enemy entities, no AC dep).
- Summon onto an adjacent tile, 1/battle (Pack → 2).
- Biomes: snow=wolves; plains=wolves/spiders; forest=+young schrats; steppe/hills=wolves/hyenas;
  desert=hyenas/snakes; swamp=spiders/snakes.
- RU beast terms verified in data_014/AC; see memory `project_druid_mod`.

## Icons - the big ongoing effort
**Convention (verified in game): perks & effects = ROUND, actives = SQUARE. A perk that grants an
active needs BOTH a round and a square icon.**

Pipeline (`tools/`, all params via env - do NOT edit the scripts):
- `icon_postprocess.sh <subj> <out> [rim]` - frameless subject on green → key green → **disc → rim
  → subject ON TOP** (subject overlaps the rim, e.g. ears/snout). env: `HEAD` size %, `DISCR` disc
  radius, `USM` unsharp, `SHP` sharpen, `SAT` saturation, `CON` contrast, `FUZZ`.
- `reframe.sh <framed_subj> <out> <rim>` - strips a baked codex frame and puts our rim on instead
  (keeps the art). env: `INNER` crop frac + the same USM/SHP/SAT/CON.
- `check_disc.py <rim> <disc>` - mechanical check: no disc past rim, no gap. Run on every icon.
- `median_frame.py <out> <icons...>` - common frame via median (use for the SQUARE active frame;
  NOT for round perks - vanilla perk frames differ).
- `ICONS.md` - settled params and per-icon status.

Settled: rim = **rim_c** (rf_follow_up ring, cut from `~/Documents/bbm/perks.xcf` → `_gen/rim_c.png`),
**DISCR=24**, sharpness **s3** (`USM=1.5x1.2+1.6+0 SHP=0x0.8`). Subjects are generated by codex on a
flat chroma green `#00d000`, frameless. **Green pockets enclosed by the subject** (between spider
legs, snout/moon, etc.) are not reached by the corner floodfill → pre-key globally first:
`magick subj.png -fuzz 26% -transparent '#00d000' subj_cut.png` then run the pipeline on the cut.
Source gens, cuts and rims live in `gfx/druid/_gen/`; old experiments in `gfx/druid/_gen/wip/`.

## Gotchas
- `cp` is aliased to `cp -i` → use `command cp -f` to overwrite.
- bash `read -r a b < <(...)` fails under `set -e` without a trailing newline → use `$(...)`.
- Disc must be ≤ the rim's inner radius or it pokes past the (thin) rim - `check_disc.py` catches it.
- Codex image gen: `codex exec --sandbox workspace-write "$(cat prompt.txt)"`, put all paths and
  instructions IN the prompt (reference images as absolute paths, not `-i`); it saves the PNG. To
  reuse an approved framed gen, have codex strip the frame+bg to a green cut-out, then pipeline it
  (that's how pack_leader was done).
